Getting an A in a programming class

  1. Attend each class, and stay for the full class
  2. Ask your Instructor to explain everything you dont understand, NEVER "let it go"
  3. Read every word in your reading assignments, and handouts
  4. Type in, and run all the programs in the handouts more than once, and with different data values, a brief read through is not enough
  5. Experiment with inventing, and writing your own programs
  6. Put any code fragments you dont understand into small programs to see what they do

If you follow these rules, you'll probably get an A.

If you DON'T, you might just get yourself an F.
